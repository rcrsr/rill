# rill control flow

Conditionals, loops, branching, and the collection-operator picker.

## Conditional

    cond ? then_expr ! else_expr
    cond ? then_expr                   # without else: returns current $ when false

Piped form (`$` becomes condition):

    value -> ? then_expr ! else_expr

Multi-line (`?` and `!` start continuation lines):

    condition
      ? "yes"
      ! "no"

    $val -> .eq("A") ? "a"
      ! .eq("B") ? "b"
      ! "c"

## Loops

Pre-condition (`while ... do`):

    init -> while ($ < 10) do { $ + 1 }   # $ is the accumulator

Post-condition (`do ... while`, body runs at least once):

    init -> do { $ + 1 } while ($ < 10)

Iteration limit (default 10,000):

    0 -> while ($ < 50) do<limit: 100> { $ + 1 }
    $items -> fan({ slow($) }, dict[concurrency: 3])

## Break and return

`break` exits a loop or `seq`/`acc` body, returning collected results so far:

    list[1,2,3,4,5] -> seq({
      ($ == 3) ? break
      $
    })                                   # list[1, 2]

`return` exits a block or the script:

    5 => $x
    ($x > 3) ? ("big" -> return)
    "small"                              # returns "big"

    "done" -> return                     # exits script with "done"

## Pass

`pass` returns `$` unchanged. Requires pipe context.

    cond ? pass ! "fallback"             # preserve $ when cond is true
    cond ? "value" ! pass                # preserve $ when cond is false
    "data" -> { dict[status: pass] }     # dict[status: "data"]
    list[1, -2, 3] -> fan({ ($ > 0) ? pass ! 0 })   # list[1, 0, 3]

## Collection operator picker

| Goal | Operator |
|---|---|
| Transform each item, sequential | `-> seq({ })` |
| Transform each item, parallel | `-> fan({ })` |
| Filter items | `-> filter({ })` |
| Single result from list | `-> fold(init, { $@ + $ })` |
| Results plus running accumulator | `-> acc(init, { $@ + $ })` |
| Stable ascending sort | `-> sort` or `-> sort({ key })` |
| Loop with state | `-> while (cond) do { }` |
| Infinite source stream | `iterate(seed, { next })` or `seed -> iterate({ next })` |
| Suppress rapid stream emissions | `-> debounce(duration)` |
| Limit to one per interval | `-> throttle(duration)` |
| Latest at fixed interval | `-> sample(duration)` |
| Bound body to wall-time limit | `timeout<total: duration> { body }` |
| Bound body to inactivity limit | `timeout<idle: duration> { body }` |

| Operator | Execution | Returns | Catches `break`? |
|---|---|---|---|
| `seq` | sequential | all body results | yes |
| `acc(init, ...)` | sequential | list of accumulator at each step | yes |
| `fan` | parallel | all body results | no |
| `filter` | parallel | matching elements only | no |
| `fold(init, ...)` | sequential | final accumulator only | no |
| `sort` | eager | sorted list, or `ordered` for dict input | no |

`$@` is the accumulator in `fold` and `acc`. See `stdlib` for `sort` details (key extractor, multi-key tuple projection, descending via `.reverse`).

## Body forms

All operators accept any callable as the body argument:

    -> seq({ $ * 2 })           # block, $ is current element
    -> seq(|x| ($x * 2))        # inline closure
    -> seq($double)             # variable closure
    -> seq(log)                 # built-in function

## Iterating dicts and strings

Dict iteration binds `$.key` and `$.value`:

    dict[a: 1, b: 2] -> seq({ "{$.key}={$.value}" })   # list["a=1", "b=2"]
    dict[a: 1, b: 5] -> filter({ $.value > 2 })

String iteration binds `$` to each character:

    "abc" -> seq({ "{$}!" })    # list["a!", "b!", "c!"]
    "hello" -> filter({ $ != "l" })

## Loop with multi-field state

Use `while ... do` with `$` as a state dict:

    dict[iter: 0, max: 3, text: $input, done: false]
      -> while (!$.done && $.iter < $.max) do {
        $.iter + 1 => $i
        process($.text) => $result
        $result.finished
          ? dict[iter: $i, max: $.max, text: $.text, done: true]
          ! dict[iter: $i, max: $.max, text: $result.text, done: false]
      }

## Iterators

Lazy sequences. All collection operators auto-expand iterators.

    range(0, 5) -> seq({ $ * 2 })      # list[0, 2, 4, 6, 8]
    repeat("x", 3) -> seq({ $ })       # list["x", "x", "x"]

`.first()` returns an iterator for any collection:

    list[1, 2, 3] -> .first()          # iterator at 1
    "abc" -> .first()                  # iterator at "a"

Iterator protocol (dict with `value`, `done`, `next`):

    $it.done                           # bool: exhausted?
    $it.value                          # current element
    $it.next()                         # iterator at next position

## Predicate-gated stream operators: `start_when` / `stop_when`

Use `start_when` and `stop_when` to slice a stream based on a predicate result rather than a fixed index.

| Goal | Operator |
|---|---|
| Drop elements until predicate first returns `true`, then yield all remaining | `-> start_when({ cond })` |
| Yield elements including the first where predicate returns `true`, then stop | `-> stop_when({ cond })` |
| Keep only elements matching a condition | `-> filter({ cond })` |
| First n elements by position | `-> take(n)` |

`start_when` vs `filter`: `filter` tests every element independently. `start_when` triggers once and then passes all subsequent elements through, regardless of the predicate.

`stop_when` is inclusive: the element that satisfies the predicate is included in the output.

Error contracts (both operators):

| Error | Cause |
|---|---|
| `#RILL_R040` | Predicate argument is not callable |
| `#TYPE_MISMATCH` | Predicate returns a non-`bool` value |

Example:

    list[1, 2, 5, 3, 4] -> start_when({ $ > 3 })   # stream[5, 3, 4]
    list[1, 2, 5, 3, 4] -> stop_when({ $ > 3 })    # stream[1, 2, 5]

## `timeout<>` blocks

Bound a body to a time limit. On expiry, the block produces an invalid value carrying a timeout atom.

| Form | Expiry atom | Trigger |
|---|---|---|
| `timeout<total: duration> { body }` | `#RILL_R082` | Wall-time from block entry |
| `timeout<idle: duration> { body }` | `#RILL_R083` | Inactivity: no stream chunk within duration |

`total` and `idle` are mutually exclusive. The duration must be a `duration` value; non-duration halts with `#INVALID_INPUT`.

Expiry produces a catchable halt. Always wrap in `guard` to catch it:

    guard { timeout<total: duration(0,0,0,0,0,0,500)> { $app.fetch($url) } } => $r
    $r ?? "cached"                                    # one-liner fallback

Direct `??` on a bare `timeout<>` does NOT catch expiry. Both `total` and `idle` work with `guard`.

## `pass` body forms

`pass` runs a side-effect block and returns the pipe value unchanged. The body form has two surfaces: with and without `<...>` options. Suppression of catchable halts requires `on_error: #IGNORE` explicitly; a bare `pass { body }` does NOT suppress halts.

Note: `pass<>` (empty angle brackets) is a parse error (RILL-P004). Always use `pass { body }` or `pass<on_error: #IGNORE> { body }`.

| Form | Purpose |
|---|---|
| `pass` | References current piped value `$`; halts #RILL_R005 if unbound |
| `pass { body }` | Runs body for side effects; pipe value unchanged; does NOT suppress catchable halts |
| `pass<on_error: #IGNORE> { body }` | Runs body for side effects; pipe value unchanged; suppresses catchable halts in body |
| `pass<async: true> { body }` | Fire-and-forget; body dispatched without blocking; pipe value returned immediately |
| `pass<async: true, on_error: #IGNORE> { body }` | Fire-and-forget; body halt suppressed |

`on_error: #IGNORE` suppresses catchable halts inside the body. Non-catchable halts (`error`, `assert`) and `ControlSignal` (`break`, `return`) still propagate.

`pass<async: true>` bodies may complete after downstream operators run. Scripts must not depend on body completion order relative to downstream execution.

Example:

    5 -> pass { log($) }                                          # logs 5, yields 5
    5 -> pass<on_error: #IGNORE> { log($) }                      # logs 5, yields 5; body halts suppressed
    "data" -> pass<on_error: #IGNORE> { audit($) } -> .upper     # "DATA"
    42 -> pass<async: true, on_error: #IGNORE> { app::log($) }   # fire-and-forget; yields 42

Do NOT use `pass { body }` when you need the body result — the body result is always discarded.

See `errors` for `assert`, `error`, `guard`, `retry`. See `callables` for closure forms.
