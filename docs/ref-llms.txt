# rill

> Scripting language designed for machine-generated code. LLMs generate rill scripts; humans review and debug. Data flows through pipes (`->`), not assignment. Variables use `$` prefix. No null, no exceptions, no truthiness.

## Rules an LLM must internalize

1. **Pipe (`->`) flows data; capture (`=>`) stores it.** No `=` operator. Statement order is left-to-right through the chain.
2. **Conditions must be boolean.** No truthiness. `"" ? a ! b` is an error. Use `.empty`, `.eq()`, comparison operators.
3. **Errors halt by default. Resolve to valid via `??`.**
4. **All variables start with `$`.** `$name` references a variable. Bare `name` is a dict key, a built-in, or a host function.
5. **Script-defined names live in variables, even callables.** A closure becomes a `$` variable: `|x|($x*2) => $double` then `5 -> $double`. Only built-in and host functions stay bare.
