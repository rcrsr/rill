# rill Extensions Index

> Discovery index for all rill extensions. Each entry lists the npm package, a summary, and a deep link to full documentation. Extensions are configured in `rill-config.json` under `extensions.mounts`.

## How extensions load

```json
{
  "extensions": {
    "mounts": { "<namespace>": "<npm-package>" },
    "config": { "<namespace>": { ... } }
  }
}
```

```rill
use<ext:namespace> => $ext
$ext.function_name(args)
```

## Bundled Extensions (shipped with @rcrsr/rill)

Bundled extensions import from `@rcrsr/rill/ext/<name>` in TypeScript host code. They do not require `rill-config.json` mounts.

| Extension | Import | Functions | Summary |
|-----------|--------|-----------|---------|
| fs | `@rcrsr/rill/ext/fs` | 12 | Sandboxed local filesystem with mount-based permissions |
| fetch | `@rcrsr/rill/ext/fetch` | per-endpoint | HTTP client with endpoint allowlisting (no arbitrary URLs) |
| exec | `@rcrsr/rill/ext/exec` | per-command | Sandboxed command execution with allowlist/blocklist |
| kv | `@rcrsr/rill/ext/kv` | 11 | JSON-backed key-value store with optional schema |
| crypto | `@rcrsr/rill/ext/crypto` | 4 | Hashing (SHA-256), HMAC, UUID, random bytes |

Documentation: https://raw.githubusercontent.com/rcrsr/rill/main/docs/bundled-extensions.md

## Vendor Extensions (from @rcrsr/rill-ext)

Vendor extensions install as separate npm packages and load via `rill-config.json` mounts. Extensions in the same category share identical function signatures, so scripts swap providers by changing the mount package.

### LLM (3 interchangeable packages, 6 shared functions)

Single-prompt, multi-turn conversation, embeddings, tool loops, structured output.

| Package | Provider | Docs |
|---------|----------|------|
| `@rcrsr/rill-ext-anthropic` | Claude and Anthropic-style APIs | https://raw.githubusercontent.com/rcrsr/rill-ext/main/packages/ext/llm-anthropic/docs/extension-llm-anthropic.md |
| `@rcrsr/rill-ext-openai` | GPT and OpenAI-style APIs | https://raw.githubusercontent.com/rcrsr/rill-ext/main/packages/ext/llm-openai/docs/extension-llm-openai.md |
| `@rcrsr/rill-ext-gemini` | Google (Gemini) | https://raw.githubusercontent.com/rcrsr/rill-ext/main/packages/ext/llm-gemini/docs/extension-llm-gemini.md |

Functions: `message`, `messages`, `embed`, `embed_batch`, `tool_loop`, `generate`

Note: Anthropic does not support embeddings. Use OpenAI or Gemini for `embed`/`embed_batch`.

### MCP — Model Context Protocol (1 package)

Connects rill scripts to MCP servers via stdio or HTTP transport. Auto-generates callable functions from server metadata. MCP tools pass directly to LLM `tool_loop`.

| Package | Transport | Docs |
|---------|-----------|------|
| `@rcrsr/rill-ext-mcp` | stdio, HTTP | https://raw.githubusercontent.com/rcrsr/rill-ext/main/packages/ext/mcp/docs/extension-mcp.md |

Returns three capability dicts: `.tools`, `.resources`, `.prompts`

### Claude Code (1 package, 3 functions)

Spawns the Claude Code CLI as a subprocess. Requires `claude` binary in `$PATH`.

| Package | Requires | Docs |
|---------|----------|------|
| `@rcrsr/rill-ext-claude-code` | `claude` CLI, `node-pty` | https://raw.githubusercontent.com/rcrsr/rill-ext/main/packages/ext/claude-code/docs/extension-claude-code.md |

Functions: `prompt`, `skill`, `command`
