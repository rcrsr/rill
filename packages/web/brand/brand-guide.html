01 — Brand Identity
Category & Purpose
rill occupies the developer tools category — specifically, embeddable scripting runtimes. It targets developers building AI agents who need a safe runtime for generated code.
->

Developer Tools — Embeddable Scripting Runtime

A sandboxed scripting runtime that host applications embed safely. rill eliminates null, exceptions, and implicit coercion. Data enters, transformations shape it, and a result emerges.

Core Metaphor

Data in motion

The rill brand embodies continuous flow. The logo's forward-leaning parallelograms suggest momentum — data entering from the left, transforming through each stage, emerging on the right. The neon spectrum traces this journey visually.

Brand Personality

Precise & luminous

rill speaks with technical precision and visual clarity. No weasel words. No ambiguity. The neon-on-void aesthetic makes every element intentional and legible. The brand feels like a well-lit terminal at 2 AM — focused, energized, alive.

Audience

Agent developers

Developers building AI agents, copilots, and automation tools. They need a scripting runtime that LLMs can generate code for and end users can execute without risk. They value safety guarantees over language flexibility.

Positioning

Sandboxed by design

Where other runtimes require sandboxing after the fact, rill has no filesystem access, no network access, no side effects unless the host explicitly provides them. Safety is the default, not a configuration option.

02d Clear Space & Sizing

Maintain minimum clear space of 1.5x the logo's bounding dimension on all sides. Minimum size for the full mark: 80px width. Minimum size for the icon: 16px.

Correct Usage
Neon logo on dark backgrounds. Maintain clear space. Let the glow breathe.
Correct — Mono on Light
Use mono black on light surfaces. Never place the neon logo on light backgrounds.
Correct — Icon at Small Size
Use the icon variant below 80px. The full mark loses legibility at small sizes.
Avoid
Neon on light backgrounds washes out. Never desaturate, rotate, or stretch any variant.

08 — Voice & Tone
How rill Speaks
rill's voice mirrors its design principles: explicit, flowing, and direct. No marketing fluff. No vague promises. Say what the language does, show code, let results speak.

Do

"Data flows left-to-right through pipes. No assignment operator exists."
"filter removes items that fail a boolean test."
"rill scripts run inside host applications that provide domain functions."

Don't

"rill is a powerful, next-generation scripting solution."
"Easily filter your data with our intuitive API."
"Supercharge your workflows with rill's amazing features."

Naming Conventions

Lowercase "rill" — never "Rill", "RILL", or "rill.js"
The pipe operator: -> — always with arrow, never "pipe symbol"
Functions: verb form — "filter", "map", "fold" — not "filtering", "mapping"

Documentation Pattern

1. State what the feature does in 1 sentence
2. Show a code example
3. Explain the result
4. Note mainstream habits to break (optional)
09 — Composition
Assembled Example
All brand elements working together — typography, color, spacing, and code styling in a realistic card layout.

Agent Scripting Runtime

Natural Dataflow

Data enters and flows through transformations via the -> operator. Expressions chain naturally. Code reads like the transformation it describes.

No Null, No Exceptions

Missing values produce errors — use ?? for defaults. No try/catch exists. Validate inputs, check outputs, let errors halt execution cleanly.

# Fetch, parse, transform, deliver app::fetch("https://api.example.com/users") -> parse_json -> filter { .role == "admin" } -> map { [name: .name, email: .email] } -> each { app::notify("Alert for {.name}") }
