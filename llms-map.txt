# rill ecosystem map

> Canonical manifest for the rill stack. Start here, then load the specific docs for what you need to do. Every link below either lives in this repo or is a stable GitHub URL under github.com/rcrsr.

## What rill is

rill is a scripting language designed for machine-generated code. LLMs author rill scripts; humans review and debug. Data flows through pipes (`->`), not assignment. Variables use `$` prefix. There is no null, no exceptions, and no truthiness.

## Rules an LLM must internalize before generating rill code

1. Pipe (`->`) flows data; capture (`=>`) stores it. There is no `=` operator.
2. Conditions must be boolean. No truthiness. Use `.empty`, `.eq()`, and comparison operators.
3. Errors halt by default. Resolve to a valid value with `??`.
4. All script variables start with `$`. Bare names are dict keys, built-ins, or host functions.
5. Script-defined names, including closures, live in `$` variables. Only built-in and host functions stay bare.
6. Strings use double quotes only. Single quotes are a parse error.
7. `+` is arithmetic-only. Use interpolation (`"{$a}{$b}"`) or `.join()` for strings.

For the expanded rules and worked examples, load [docs/llm/cheatsheet.txt](docs/llm/cheatsheet.txt).
